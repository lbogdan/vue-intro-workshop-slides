Two-way Property Binding: v-model on DOM elements

  • variable is a property of state (can go deeper, e.g. user.name)

Methods


            
            
            <script>
            export default {
              methods: {
                greet() {
                  alert('Hello, ' + this.name + '');
                },
              },
            };
            </script>
  • inside the method this is auto-bound to component instance
  • accessible in template as greet() (or just greet for event handlers)
  • accessible in script as this.greet()
Intro to Vue.js

DOM Event Handling: v-on on DOM elements

  • syntax: v-on:event="handler"
  • shorthand: @event="handler"
  • event is the DOM event name (e.g. click)
  • handler is a method name defined in methods
  • the event handler is called with the DOM event as an argument
Intro to Vue.js

Event Handling: v-on (cont.)

  • alternate syntax: @event="statement"
  • statement can be any valid JavaScript statement
  • only use simple statements!
  • modifiers: @submit.prevent="save": prevent, once

Style Binding

  • syntax: :style="object_expression"
  • object_expression is a JS expression that evaluates to an object
  • object properties are CSS properties (can be both camelCase and kebab-case)
  • object values are CSS values

Challenge 7!

Intro to Vue.js

Derived Component State: computed

  • a state property that only depends on other state properties

            export default {
              computed: {
                fullName() {
                  return this.name + '.' + this.lang;
                },
              },
            };
  • accessible in template as fullName (without ()!)
  • accessible in script as this.fullName (same as data)
  • rule of thumb: if it's internal state, but it can't be modified, it's a computed
  • cached!

Challenges 13-15!

Intro to Vue.js

Two-way Property Binding: v-model on components

  • syntax: <custom-input v-model="variable">
  • eqivalent to :value="variable" @input="variable = $event"
  • $event is a special Vue property that holds the event payload
  • variable is a property of state
  • the child component must accept a value prop and can emit input events with the new value
Intro to Vue.js

Reacting to state changes: watchers


            export default {
              watch: {
                name(value, oldValue) { // called when this.name changes
                  console.log(`name changed from ${oldValue} to ${value}`);
                },
              },
            };
  • if we need to watch more than one level, we can use deep watchers

Component Life-cycle Hooks (not that kind of hooks! :-) )

  • called during component life-cycle
  • most used ones:
    • created() - called after the instance is initialized, but before render
    • mounted() - called after rendering and attaching to DOM
    • beforeDestroy() - called before destroying the instance, do cleanup tasks here (e.g. delete timers)

              export default {
                created() {
                  console.log('instance created!');
                },
              };
  • are synchronous (Vue doesn't wait for them to "finish")
